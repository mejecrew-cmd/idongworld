# 완료 기록 2026-05-29

아래 내용은 완료된 작업 보드의 원문 기록이다. 새 작업 계획으로 사용하지 않는다.

기존 2026-05-24 작업 보드의 1~8번은 2026-05-29 기준으로 완료됐다. 자세한 완료 기록은 관련 runbook과 계획 문서의 changelog를 따른다.

## 현재 완료 상태

- backend는 단일 Express 서버를 유지한다.
- MongoDB 연결과 local 우선 개발 기준이 정리됐다. Atlas는 사용 여부 미정으로 보류 기준만 문서화했다.
- `/api/state` 통합 state route는 제거됐다.
- frontend sync는 account, host, module 전용 API를 사용한다.
- account, host, my-aidong, my-island, codex, route-neighbor, ship, zone 계열은 전용 document/repository 경계를 가진다.
- module state API와 module action API가 `/api/modules/{moduleId}` 기준으로 정리됐다.
- customs는 registered rule, resource adapter, audit log, idempotency, rollback 보강을 가진다.
- zone collect/clear는 backend authority action으로 이동했다.
- codex action validation은 backend service에서 처리한다.
- production auth fallback 차단과 Firebase Bearer token 경로가 준비됐다.
- frontend store는 물리 분리하지 않고 단일 Zustand `userStore`를 유지하며, `storeFacades.ts`를 우선 사용하기로 결정했다.

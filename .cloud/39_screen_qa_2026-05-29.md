## 2026-06-01 Playwright Smoke 결과

브라우저 클릭 자동화를 1차 도입했다.

추가 파일:

```txt
playwright.config.ts
tests/e2e/app-smoke.spec.ts
```

추가 명령:

```bash
pnpm check:e2e:smoke
```

확인 범위:

- backend `/health`가 정상이고 Mongo repository를 사용한다.
- `/` 진입 시 `/login`으로 이동한다.
- `게스트로 시작` 클릭 후 `/title`로 이동한다.
- `/island`, `/island/lodge`, `/island/harbor`, `/island/full-map`, `/voyage/board?route=neighbor`, `/dev/catalog` route가 브라우저에서 열린다.
- `/island/harbor`와 `/island/lodge`에는 `배 세관`, `숙소 세관` 편의 버튼이 없다.
- 정원 미니게임을 닫을 때 `나가기 전 세관` 팝업이 열린다.
- 위 흐름 중 `/api/state` 요청이 발생하지 않는다.
- page runtime error가 발생하지 않는다.

실행 결과:

```txt
pnpm check:e2e:smoke
1 passed
```

남은 수동 확인:

- first gacha와 onboarding 전체 연출은 아직 Playwright smoke에 넣지 않았다.
- lodge Aidong item 착용, route landing clear, ship slot UI는 별도 E2E로 확장한다.
- customs UI는 팝업 진입까지 확인했다. 실제 inventory 변환 성공/실패는 다음 E2E 확장 대상으로 둔다.

## 2. 게임기획 구조 분석

회의 내용을 게임 시스템 레이어로 재배열했다. 각 레이어에서 **현행 기획 → 회의 결정**의 골자를 정리한다.

### 2-1. 코어 게임 루프 (Core Loop)
- **정체성 재확인**: 표면은 "아이동을 데려다 연습시켜 아이돌로 만든다"지만, **실체는 베니월드(VVNS)의 재활용 = 생활형 육성/수집/배치**다([33:23]~[34:09]). "아이돌 만들기"는 제안서의 구색.
- **확정된 루프 골격**([1:02:40]~[1:03:57], [42:23]):
  1. **항로(항해)** 에서 ① 아이동을 만나고 ② 재료(아이템)를 수집
  2. 만난 아이동은 **메인 화면에 전용 구역**으로 추가됨 + **숙소(인벤토리)** 에 합류
  3. **숙소**에서 케어·연습·꾸미기 + 남는 아이동은 각 **구역/모듈에 배치**해 자원 자동 생산
  4. **도감 아이템**을 모아 아이동 **업그레이드**(댄스머신·랩머신 등)
  5. 배 적재량이 차면 **입항해 비우고** 다시 항해 → 반복
- **핵심 동사**: 산다 / 발견한다(항해) / 곁에 둔다(구역·숙소) / 키운다(연습·업그레이드). 기존 정체성 문서의 "산다·발견한다·곁에 둔다"와 정합하나, **"키운다(업그레이드 경제)"가 회의에서 구체화**됨.

### 2-2. 아이동 만남·영입·구역 시스템 ★최대 변경
- **현행**: 항로 보드 위에 *아이동 섬*이 슬롯으로 배치되고, 방문→영입하면 그 섬은 역할을 잃음(일회성 문제)([45:42]~[46:25]).
- **회의 결정**: 아이동을 만나면 **메인 화면에 "그 아이동 구역"이 슬롯처럼 하나씩 붙는다**([00:00]~[01:41], [46:25]~[51:53]). 만남은 랜덤, 만나는 **순간 오프닝 스토리 1컷 → 메인에 구역 개방**.
  - 섬을 "버리는" 게 아니라 **메인 구역으로 흡수**되는 구조 → 일회성 문제 해소.
  - 구역은 계속 늘어나며, 많아지면 **이별하기(빼기)**로 정리([51:37]~[51:53]).
  - 각 아이동 구역에는 **지정 미니게임 1종 + 스킨**이 붙는다([47:38]~[48:45], [49:14]~[50:17]).
- **미니게임 = 스킨 차이**: 예) 황금멍·찌린양이 둘 다 "카드 뒤집기"여도, 리소스/이미지/획득 아이템만 종별로 다른 **같은 게임의 스킨**([48:45]~[50:17]). 766마리를 개별 게임으로 만들지 않기 위한 절약 설계.

### 2-3. 인벤토리·자원·재화 경제 ★최대 변경
- **현행**: 모듈별 로컬 자원 ↔ 글로벌 자원을 **세관(Customs)** 으로 변환. 인벤토리는 "1번 슬롯=밧줄 고정" 식 슬롯 고정 모델 검토([18:27], [17:44]).
- **회의 결정**:
  - **세관 강제 변환·1번 슬롯 고정 → 폐기/보류**. "**통으로** 70만 개 인벤토리를 두고 1번부터 필요한 만큼 키운다"([57:57]~[58:30]).
  - 배 인벤토리 / 내 인벤토리 **구분 없음**(세관을 안 거치므로)([56:18]~[56:40]).
  - **아이동이 자기 25개 도감 아이템 인벤토리를 보유**(광역 데이터). 아이동 1~5명이면 25×5 인벤토리. 항로에서 얻은 도감 아이템은 **배가 아니라 해당 아이동에게 귀속**([53:54]~[55:35]).
  - 항해 공통 아이템(조개=화폐 등)은 **배 화물칸**에 적재([06:49], [07:44]).
- **모듈화의 게임적 정당화는 유지**: "아이동과 이별하면 그 25 인벤토리가 빠져나가 생산 중단" = 사실상 **아이동 단위 로컬화**가 세관 없이도 성립([56:40]~[57:49]). → 세관이라는 *기술 장치* 없이 *게임 규칙*으로 같은 효과.

### 2-4. 숙소·배치·자동 생산 (Idle/배치 경제)
- **숙소 = 육성의 핵심 영역**. 다른 모든 활동은 숙소 업그레이드를 위해 존재([31:30]).
- **방 = 영입(육성) 제한, 마당 = 보유 인벤토리 표시**([34:15]~[38:23]):
  - 방이 2개면 적극 육성 슬롯 2 → 증축으로 확장([36:15]~[36:35]).
  - 나머지는 마당에 표시(= 인벤토리). "마당"은 표현일 뿐 **숙소에서 전체 아이동을 컨트롤**([42:23]~[43:52]).
- **아이동 배치 = 자원 자동 생산**: 광산 등 구역/모듈에 아이동을 배치하면 시간당 아이템 자동 생산([36:43]~[37:46], [52:34]~[53:11]). 아이동 속성/케미에 따라 효율 차등(빨간 마크=비효율)([37:33]~[37:46]).
- **구분된 두 트랙**: ① 보석함 아이동(숙소·연습실로 육성) ② 부려먹는 아이동(광산·등대 등 배치 생산)([43:52]~[44:21]).

### 2-5. 성장·업그레이드 (도감 = 성장 재료)
- **도감 아이템 25종/아이동 = 업그레이드 연료**([55:35]~[56:18], [1:00:04]~[1:01:24]):
  - 숙소 연습 + 도감 아이템 충족 → 댄스머신/랩머신 등으로 단계 업그레이드.
  - 베니월드식 "모으기만 하던 도감"을 **업그레이드 소비처로 전환**.
- **크래프팅(제조)**: 마인크래프트식 레시피 제작. 재료 추적(어디서 얻는지) + **아이돌 관련 물품(마이크 등) 제작**([53:11]~[53:54]).
- **친구 간 교환**: 같은 아이동 보유자끼리 아이템 교환 가능성([1:01:03]~[1:01:24]) — Post-Proto 후보.

### 2-6. 항해·보드 (탐험)
- **모델 = 바다판 부루마블 + 대항해시대**([06:49]~[08:31], [44:38]~[45:42]). 항구=도시, 아이템 매매, 항구별 거주 아이동 영입 시나리오.
- **주사위 복잡도 재검토**: 주사위(시간/돈 소모)는 첫 사용자에게 어렵다는 반론([12:53], [30:35]~[31:30]). → "**주사위를 굴리지 말까**", 칸을 8~9칸으로 줄이거나 지정 이동을 옵션으로([12:53], [12:28]). **주사위 채택 여부 미결**.
- **항로 파밍 복귀 문제**: 곡괭이가 필요해 파밍하러 갔는데 정작 들고 나오지 못하는 동선이 어렵다는 핵심 UX 지적([30:35]~[31:30]). 세관 폐기로 일부 해소.
- **배 적재량 = 창고/입항 사이클**: 적재량 한계 → 입항해 비우고 재출항. 배 업그레이드로 적재량·선실 확장([1:01:24]~[1:02:40]).

### 2-7. 데이터 아키텍처 결정 (모듈화) ★최대 변경
- **현행 헌법(모듈분리작업계획 v1.1)**: 5분류(시스템·콘텐츠·광역·자산 풀·공유 데이터 풀) + **세관(Customs)** 변환 + 모듈 간 직접 import 금지 + DI.
- **회의 결정**:
  - **기획적 모듈화(세관 강제)는 현 단계 불필요 → 보류**. "통으로 가자"([57:49]~[59:07]).
  - **단, 데이터 카테고리 구분은 유지**: MongoDB 단일 + documents/카테고리로 이미 구분됨([26:04]~[26:33], [59:29]). "모듈이 된다고 해서 데이터상 달라지는 건 없다."
  - **백엔드 모듈/게임별 API 분리는 이미 되어 있고 유지**([23:39], [24:04], [26:04]~[26:24]).
  - **리팩토링(본격 모듈화)은 올해 말~내년 초로 미룸**. 필요 없을 수도 있음([58:30]~[58:42]).
  - **전용 아이템/완전 모듈화의 트리거 = 외부 아이돌 IP 콜라보 계약 시점**([40:20], [58:42]~[59:07]).
- **⚠️ 해석 주의**: "통으로"는 **세관 추상화를 게임 로직·인벤토리에 강제하지 말자**는 뜻이지, **이미 분리된 백엔드/카테고리 구조를 헐자는 뜻이 아니다.** 대표(김동은)는 기술 지도를 하지 않으며, 통/분리 판단은 기술팀(이성민·정윤호)이 정한다([24:31]~[25:41]).

### 2-8. 플랫폼·UI (웹 특성)
- **웹 = 정보 수집·뎁스 수용에 최적**([1:03:57]~[1:05:27]). 좁은 모바일 화면 대신 **깊고 풍부한 페이지 뎁스 허용**.
- **세션 유일성 없음**: 같은 계정 멀티탭 동시 접속 허용. 게임처럼 막지 않음. 복사버그만 차단([1:06:11]~[1:09:26]).
- **딥링크는 목표 아님**: 기왕이면 라우트 분리가 좋으나 프로그램 제약을 따름([1:05:27]~[1:07:05]).
- **인사이클로피디아(백과사전)**: 766마리 도감은 **로드맵/먼 미래**. 초기엔 만들 수 있는 20~30마리를 **전부 랜덤**으로 등장([03:25]~[05:35], [58:25]).
- **BM 보류**: 콘서트 예매·포도알 시스템은 "가치 아이템을 쉽게 얻게 한다"는 우려로 **현 단계 미도입**([1:09:01]~[1:09:26]).

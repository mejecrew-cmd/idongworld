## 한 줄 정의

아이동월드 Phase 1의 핵심 루프는 **항해에서 아이동과 재료를 발견하고, 숙소에서 아이동을 돌보고 키우며, 각 구역에 배치해 자원을 생산하고, 도감 아이템으로 업그레이드한 뒤 다시 항해하는 생활형 육성 루프**다.

세관과 완전 모듈화는 이 루프의 필수 사용자 행동이 아니다. 기술 자산으로 보존하되, Phase 1 화면에서는 숙소와 통 인벤토리 중심 경험을 우선한다.

## 핵심 동사

| 동사 | 의미 | 주요 화면 |
|---|---|---|
| 산다 | 숙소에서 아이동과 지내고 케어한다 | 숙소 |
| 발견한다 | 항해에서 아이동, 재료, 도감 아이템을 찾는다 | 항구, 항해 |
| 곁에 둔다 | 아이동을 숙소, 마당, 메인 구역에 배치한다 | 메인, 숙소, 구역 |
| 키운다 | 케어, 연습, 업그레이드로 성장시킨다 | 숙소, 연습/업그레이드 |
| 부린다 | 남는 아이동을 구역에 배치해 자동 생산한다 | 가변 구역, 생산 구역 |
| 비운다 | 배 적재량이 차면 입항해 정리하고 다시 떠난다 | 항구 |

## Phase 1 우선 루프

```txt
1. 항구에서 항해를 시작한다.
2. 항해 중 아이동 또는 재료/도감 아이템을 발견한다.
3. 아이동을 만나면 짧은 오프닝 스토리 1컷을 본다.
4. 만난 아이동은 숙소에 합류하고 메인 화면에 전용 구역 슬롯을 연다.
5. 숙소에서 아이동을 케어, 연습, 꾸미기, 업그레이드한다.
6. 숙소 방 수는 적극 육성 가능한 아이동 수를 제한한다.
7. 남는 아이동은 마당 또는 구역 배치 후보로 보여준다.
8. 구역에 아이동을 배치하면 시간당 자원을 자동 생산한다.
9. 항해와 생산으로 모은 도감 아이템을 아이동 업그레이드에 사용한다.
10. 배 적재량이 차면 항구에 입항해 비우고 다시 항해한다.
```

## 이전 루프와의 차이

| 이전 POC 루프 | 새 Phase 1 루프 |
|---|---|
| 도착 섬에서 전용 자원 획득 | 항해에서 아이동/재료/도감 아이템 발견 |
| 섬을 떠나기 전 세관 필수 | 세관은 사용자 필수 gate가 아님 |
| destination island -> ship -> lodge | 통 인벤토리와 아이동 귀속 인벤토리 중심 |
| 배 인벤토리와 숙소 인벤토리 분리 | 배/내 인벤토리 경계는 재설계. 구분 없음 방향 |
| 아이동 섬이 항로 위 목적지 | 만남 후 메인 가변 구역 슬롯으로 흡수 |
| 4개 zone 미니게임 고정 | 아이동별 지정 미니게임 + 스킨 |
| 도감은 등록/수집 중심 | 도감 아이템은 업그레이드 연료 |

## 화면별 역할

### Title/Login/Opening

- 기존 로그인과 최초 계정 생성 흐름은 유지한다.
- opening은 최초 계정 생성 또는 최초 만남 같은 명확한 계기에서만 보여준다.
- 이미 로그인된 사용자가 매번 opening을 반복해서 보지 않도록 한다.

### 메인 화면 / 마이섬

- 기본 구역과 아이동 전용 가변 구역 슬롯을 함께 보여주는 허브다.
- 아이동을 만나면 해당 아이동의 전용 구역 슬롯이 추가된다.
- 고정 15구역과 가변 구역의 공존 방식은 P0 미결이다.
- 메인 화면은 숙소, 항구, 생산 구역으로 자연스럽게 이동할 수 있어야 한다.

### 항구

- 항해 시작과 입항/비우기의 중심 화면이다.
- 배 적재량, 현재 항해 상태, 출항 가능 여부를 보여준다.
- 배 업그레이드는 적재량과 선실 확장 후보로 둔다.
- 세관 버튼을 핵심 버튼으로 세우지 않는다.
- 배 적재량과 입항/비우기 사이클의 상세 기준은 `.cloud/62_ship_cargo_port_cycle_2026-06-08.md`를 따른다.

### 항해

- 아이동 만남, 재료 획득, 도감 아이템 획득을 담당한다.
- 주사위/칸 수/지정 이동 방식은 P0 미결이다.
- 30칸 주사위 보드 고도화는 보류하고, 결정 전까지 현재 구현은 상태 저장/복귀 안정성 위주로 유지한다.
- landing은 `Aidong encounter`, `material gain`, `codex item gain`, `event` 중심으로 재설계한다.
- 항해 보드 단순화 재논의 자료는 `.cloud/61_voyage_board_simplification_2026-06-08.md`를 따른다.

### 숙소

- Phase 1의 핵심 육성 허브다.
- 방은 적극 육성 슬롯이다.
- 마당은 보유 아이동 또는 대기 아이동을 보여주는 표현이다.
- 숙소에서 케어, 밥주기, 연습, 꾸미기, 업그레이드, 아이동 목록 관리를 연결한다.
- 방 꾸미기는 유지하고 확장한다.
- 숙소 인벤토리는 통 인벤토리 설계 이후 역할을 다시 정한다.
- 숙소 허브의 상세 설계는 `.cloud/57_lodge_growth_hub_2026-06-08.md`를 따른다.

### 가변 아이동 구역

- 아이동을 만나면 메인 화면에 생기는 슬롯형 구역이다.
- 구역에는 해당 아이동의 스킨이 붙은 미니게임 또는 자동 생산 활동이 연결된다.
- 같은 미니게임 엔진을 여러 아이동이 공유할 수 있다.
- 아이동과 이별하면 해당 구역과 생산 라인이 빠지는 모델을 후보로 둔다.
- 항해 만남에서 숙소 합류와 가변 구역 추가로 이어지는 상세 계약은 `.cloud/54_aidong_encounter_dynamic_zone_2026-06-08.md`를 따른다.
- 아이동별 미니게임 스킨 모델은 `.cloud/59_aidong_minigame_skin_model_2026-06-08.md`를 따른다.

### 생산 구역

- 아이동을 배치하면 시간당 자원을 생산한다.
- 아이동 속성, 케미, 효율에 따라 생산량이 달라진다.
- 비효율 배치는 UI에서 빨간 표시 또는 경고로 알려준다.
- 기존 zone collect/clear API는 자동 생산 모델로 재사용할 수 있다.
- 구역 배치 자동 생산 상세 설계는 `.cloud/58_zone_placement_production_2026-06-08.md`를 따른다.

### 도감/업그레이드

- 아이동별 25개 도감 아이템 세트를 가진다.
- 도감 아이템은 단순 등록물이 아니라 업그레이드 재료다.
- 업그레이드는 댄스머신, 랩머신 같은 성장 장치 또는 단계 상승으로 표현한다.
- 업그레이드 결과는 생산 효율, 미니게임 보상, 케어 효율, 외형 연출, 스토리/도감 해금 후보로 나눈다.
- 통 인벤토리와 아이동 귀속 도감 인벤토리의 상세 경계는 `.cloud/55_inventory_codex_item_redesign_2026-06-08.md`를 따른다.
- 아이동별 도감 아이템과 숙소 업그레이드 경제의 상세 기준은 `.cloud/60_codex_upgrade_economy_2026-06-08.md`를 따른다.

### 세관 / Destination Island

- Phase 1 핵심 루프에서 세관은 필수 사용자 행동이 아니다.
- `CustomsTransferDialog`, `destination-shell-island`, PixiJS/Rive scene은 삭제하지 않고 POC/이벤트 섬/향후 리팩토링 후보로 보존한다.
- 외부 IP 콜라보 또는 전용 아이템 시스템이 필요해지면 세관/완전 모듈화를 다시 검토한다.

## 보류 루프

다음 루프는 당장 삭제하지 않지만 Phase 1 우선 구현에서 내린다.

- destination island에서 전용 자원 획득.
- local resource를 customs로 ship/lodge/host에 이동.
- ship inventory와 lodge inventory를 엄격하게 분리.
- 항구/숙소/섬 이탈 시 세관 팝업을 필수 gate로 여는 UX.
- 외부 IP 콜라보 전용 아이템의 완전 모듈화.

## 화면 구현 우선순위

1. 숙소 허브: 아이동 목록, 방/마당, 케어, 연습, 꾸미기, 업그레이드 진입.
2. 메인 가변 구역: 아이동 만남 후 구역 슬롯 추가.
3. 항해 만남/수집: 아이동 encounter와 재료/도감 아이템 획득.
4. 통 인벤토리: host inventory 중심의 보유 아이템 표시.
5. 구역 배치 생산: Aidong 배치와 시간당 생산.
6. 항구 입항/비우기: 배 적재량과 재항해 사이클.
7. destination island/PixiJS/Rive: 이벤트 섬 또는 연출 POC로 보존.

웹 UI/UX의 상세 방향은 `.cloud/63_web_ui_ux_direction_2026-06-08.md`를 따른다.

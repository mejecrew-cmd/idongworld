## 세관 UI 방향

- 세관 팝업은 배/숙소 메인 화면에 항상 노출되는 임의 버튼이 아니다.
- 전용 자원/아이템을 가진 모듈을 벗어나기 전에 뜨는 출구 게이트다.
- 모듈 전용 자원은 해당 모듈 바깥에서 사용할 수 없다.
- 모듈을 나가려면 전용 인벤토리를 먼저 customs rule로 변환해야 한다.
- 전역 보관소로 가는 아이템은 목적지가 `destination-island`여도 이동 가능하다.
- 갑판 적재/배로 옮기기 rule은 배를 타고 도달한 `destination-island` 계열 구역에서만 보여준다.
- 나의 섬 안의 구역을 떠날 때는 전역 보관소로 정리하는 rule만 보여준다.
- ship/lodge inventory 간 이동 UI는 추후 별도 화면으로 설계할 수 있지만, 현재 1차 UI는 "모듈 이탈 전 변환"에 둔다.

## PixiJS 부분 도입 방향

- PixiJS는 앱 전체 UI 엔진으로 도입하지 않는다.
- PixiJS는 항해 도중 도착하는 섬처럼 시각적 연출이 중요한 화면에만 부분적으로 사용한다.
- 우선 적용 대상은 `destination-island` 고정맵 탐험 화면이다.
- 각 섬 지역은 고정 배경 위에 전경/중경/후경 layer, hotspot, particle/effect layer를 얹는 2.5D scene으로 구성할 수 있다.
- 상하좌우 이동은 React route 전환이 아니라 Pixi scene 안의 node transition으로 연출하고, 필요 시 React state/router와 동기화한다.
- 미션창, 세관 팝업, 인벤토리, 방향 버튼, confirm dialog는 React/MUI overlay로 유지한다.
- backend action API 호출과 저장 상태 반영은 React screen/container가 담당하고, Pixi scene은 연출과 입력 이벤트 전달에 집중한다.
- performance budget을 두고 모바일에서 particle/filter/blur를 제한한다.

## Rive 애니메이션 방향

- 현재 기획의 destination island는 Aidong이 직접 이동하는 화면이 아니라, 고정맵의 오브젝트를 마우스로 클릭하며 상호작용하는 화면이다.
- 따라서 Spine 기반 캐릭터 이동/전투/채집 모션은 현재 단계에서 필요하지 않다.
- 캐릭터 반응, 오브젝트 반응, UI 보상 연출은 Rive를 우선 사용한다.
- Rive는 숙소/선실 Aidong idle, 케어 반응, 감정 표현, 보상 아이콘, 세관/미션 버튼, hotspot hover/click 반응에 적합하다.
- Live2D는 클로즈업 감정 표현 품질은 좋지만 작업량이 커서 현재 단계에서는 보류한다.
- PixiJS는 섬 장면의 레이어/파티클/전환 연출을 맡고, Rive는 짧은 상태 기반 반응 애니메이션을 맡는다.
- Rive state machine 입력은 React component가 제어하고, backend action API 호출은 React container가 담당한다.

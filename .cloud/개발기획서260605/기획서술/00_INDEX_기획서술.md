# 기획서술 인덱스

이 디렉토리는 기존 모듈 명세를 개발 티켓이 아니라 **기획자의 문장**으로 다시 풀어쓴다. 목적은 기능 목록을 늘어놓는 것이 아니라, 각 모듈이 왜 필요한지, 어떤 감정을 만들고, 어떤 설계 선택이 좋은지, 어디를 조심해야 하는지 리뷰 가능한 원고로 남기는 것이다.

## 작성 원칙

- 최신 결정은 `09_미결정사항_결정로그.md`를 따른다.
- 오래된 문서의 미결 표기는 이력으로만 본다.
- 각 원고는 기능, 의도, 플레이어 경험, 콘텐츠 제작 관점, 개선점, 최선점을 분리한다.
- "좋다/나쁘다"보다 "왜 이 선택이 이 게임에 맞는가"를 설명한다.
- POC v3는 온보딩, 항해 1회, 편입 1회, 케어, 도감템, 1차 성장, 데뷔, 포토카드까지를 증명 범위로 본다.

## M01. 로그인 (Auth / 계정 진입)

**목적**: 게임 진입 전 인증·동의. **소셜 로그인 전용**(ID/비밀번호 없음).

**화면 구성**
| ID | 화면 | 내용 | 상태 |
|---|---|---|---|
| AUTH-SPLASH | 스플래시 | 로고·버전 체크·**자동 로그인** 토큰 확인 | ✅ |
| AUTH-LOGIN | 로그인 | 소셜 버튼 N종 + 자동 로그인 토글 | 🟢 |
| AUTH-TERMS | 약관 동의 | 필수(이용약관·개인정보) + **연령 정보 체크** + 선택(마케팅) | 🟡 |
| AUTH-TOS | 이용약관 전문 | 약관 본문 | 🟡 |
| AUTH-PRIVACY | 개인정보 처리방침 | 본문 | 🟡 |
| AUTH-LOGOUT | 로그아웃 | 진입 전 안전 이탈·계정 전환 | 🟡 |

**세부 — 소셜 로그인 목록**
| 제공자 | 상태 | 비고 |
|---|---|---|
| 구글(=유튜브) | ✅ | 유튜브 로그인 = 구글 OAuth |
| 애플 | ✅ | iOS·Safari |
| 트위터(X) | ✅ | K-POP 팬덤 1순위 |
| 카카오 / 네이버 | 🟡 포함 | 국내 |
| 게스트(즉시 시작) | → M02 | 로그인 없이 **오프닝(튜토리얼) 시작**, 로그인·바인딩은 이후(튜토리얼과 함께 설계) |
| LINE | ✅ | JP·TW·TH·ID 팬덤 (확정) |
| Discord / Facebook | ⚪ | 글로벌 커뮤·폴백 |
| 이메일/비밀번호 | ❌ | 소셜 전용 |

**규칙·결정**
- ❌ 비밀번호·아이디 찾기 없음 (소셜 전용)
- 로그아웃 = 로그인 모듈(진입 전 이탈) / 연동·전환·탈퇴 = M19
- 자동 로그인 = 토큰 유지 / **연령 정보 체크는 약관 단계에 포함**(상세 age-gate·법정대리인 동의는 추후 검토)

## M02. 오프닝 (온보딩)

**목적**: 플레이어 → 플레이어캐릭터 다이브 + 첫 아이동 영입 + 회색섬→하트섬 전환. **1회성·단방향**.

**화면 구성**
| ID | 화면 | 내용 | 상태 |
|---|---|---|---|
| OP-01 | 현실 방 | 새벽·방·최애 포스터·다이브 | 🟢 |
| OP-02 | 바다·조각배 | 윤슬·회색섬·**선택 분기** | 🟢 |
| OP-03 | 어푸어푸 구조 | 첫 아이동(병아리형) 영입 가챠 1 | 🟢 |
| OP-04 | 소개·상륙 | 아이동 섬 안내·상륙 | 🟢 |
| LAND-01 | 첫 상륙 지도 | 15구역 첫 노출(회색·구름) | 🟢 |
| LAND-02 | 숙소 재건 전 | 폐허·청소 퀘스트 | 🟢 |
| LAND-03 | 15구역 둘러보기 | 구역 클릭 퀘스트 | 🟢 |
| LAND-04 | 이름 짓기 | 숙소명·섬명 저장 | 🟢 |

**세부 — 분기 & 상태값**
| 항목 | 내용 |
|---|---|
| OP-02 분기 | `[어푸어푸 살펴본다]` / `[큰 섬으로 돌진]` → 둘 다 구조로 수렴(돌진 시 추격 대사) |
| 첫 가챠 | 1마리 고정, **retry 없음** |
| 상태값 | firstRun · openingBranch · visitedZones · lodgeBuilt · lodgeName · islandName |

**규칙·결정**
- 단방향 1회 한정 / 어푸어푸 = 첫 아이동(고정)
- 오프닝 = 시스템 설명 아님, "플레이어→PC 진입 장치"
- **게스트 오프닝 시작**: 로그인 없이 오프닝(튜토리얼) 바로 진입 → 소셜 로그인·계정 바인딩은 오프닝 이후(저장 시점). 게스트 흐름은 튜토리얼과 함께 설계(추후)

**미결·쟁점·고려** (상세 → [모듈상세_M02_오프닝_샘플_260608.md])
- 🟠 **미결**: 첫 아이동 최종 인물 · 4시간 의식 길이 · 게스트 바인딩 시점 · 컬러 복구 연출
- ⚔️ **쟁점**: 4시간=리텐션 vs 진입장벽 / 첫 아이동=고정 병아리형 타입 안에서 최종 인물 확정
- 💡 **고려**: 이름 욕설필터·길이제한 · 복귀 알림 강도 · 1회 한정 vs 변형 의식 · 게스트 데이터 유실 방지 · IP 노출 룰

## 부록. 모듈 간 정합 메모

- **M04 숙소(장소) ──꾸미기──▶ M12 마이룸 꾸미기(모듈)**: 연결 관계(중복 아님). 숙소=거주·케어, 마이룸=꾸미기 기능. 명칭만 일원화 검토.
- **M01 로그인 ↔ M19 설정**: 로그아웃=M01 / 계정관리=M19로 분리 확정.
- **M05 항해 ↔ M06 구역**: 항해로 만남 → 구역 슬롯 편입(절차=별도 문서).
- **미결 잔여**: age-gate 상세, 첫 아이동 최종 인물, 게스트 바인딩, 수치·밸런싱. 게스트 로그인 → **M02 튜토리얼 연계로 이관**(추후 설계).

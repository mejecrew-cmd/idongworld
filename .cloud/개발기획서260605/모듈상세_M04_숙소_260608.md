# 🧩 M04. 숙소 (SOOKSO) — 상세 명세

> **위상**: 메인맵 **2구역(숙소 부지) 내부**. 마이섬=외부 지도, 숙소=내부 거주·**케어·육성 거점**.
> 줌인 다이오라마. **케어 대시보드가 사는 곳**(M03 H-1 확정). 무대는 여기 아님(=구역, M06/M08).
> **상태**: 🟢 구현 · ✅ 확정 · 🟡 기획중 · 🟠 미결

---

## 0. 개요
- **정체**: 메인맵 2구역 클릭 → **숙소 내부(줌인)**. 사이드뷰 다이오라마(VVNS House.jsx 직이식).
- **기능**: ① 일상 케어(다마고치) ② 육성(연습실·GYM 등 실내 편의시설) ③ 보유 아이동 관리(방·마당) ④ 꾸미기.
- **핵심(6/5)**: 숙소 = 육성의 핵심 영역. 모든 활동이 숙소 업그레이드를 향함.

## 2. 화면별 상세 명세

### LODGE-MAIN · 숙소 내부 (다이오라마)
- **영역**: 마이섬 2구역 내부
- **목적**: 거주·케어·육성·관리의 허브. 방 클릭 → 그 방 내부 줌인.
- **진입**: 메인맵 숙소 구역 탭 / 하단탭 `육성`
- **UI 요소**: 사이드뷰 다이오라마(전·중·배경 3레이어) · 줌 인/아웃 · 방·시설 클릭 · 보유 아이동 표시(방+마당)
- **이탈→다음**: 방→케어 / 연습실·GYM→육성 / 주방·재봉틀→크래프팅 / 마당→배치(M06)
- **데이터**: `Lodge`(§3)
- **상태**: 🟢 (VVNS House.jsx) / 🟡 (신규 시설)

### LODGE-ROOM · 방 (= 육성 슬롯)
- **목적**: 적극 육성 슬롯. **방 수 = 영입(육성) 제한**.
- **구성**: 호스트 방(인벤토리·스케줄·일기) + 아이동 방(의상·시그니처·우연 케어)
- **확장**: 방 추가 = 짓기 이벤트(자재+시간) / **저택 등급업 시 방 수 증가**
- **상태**: ✅

### LODGE-YARD · 마당/갑판 (= 인벤토리)
- **목적**: 방 못 받은 보유 아이동 표시 = 인벤토리. **숙소에서 전체 아이동 컨트롤**.
- **동작**: 마당의 아이동 → 광산·등대 등 **구역에 배치(M06)** 로 보내 생산
- **상태**: ✅ (6/5)

### 실내 편의시설 (저택 등급별 구성) 🟡
| 시설 | 기능 | 모듈 |
|---|---|---|
| 연습실 | 보컬·댄스 훈련 | M08 |
| GYM(헬스장) | 컨디션·스탯 훈련 | M08 |
| 주방 | 음식 크래프팅·밥 먹이기 | M07·크래프팅 |
| 재봉틀(공방) | 의상 크래프팅 | M13 |
| 욕실 | 청결(씻기기) | M07 |
| 거실 | 본진 모임·풀샷 모드 | M03 |
| 수영장·목욕탕 등 | 휴식·고등급 한정 | Phase 2~3 |

### LODGE-DECO · 꾸미기 진입 (→ M12 마이룸 꾸미기)
- **목적**: 숙소에서 **꾸미기 모듈(M12) 진입점**. 실제 가구·벽지·바닥·분위기·타유저 방문 기능은 **M12가 담당**.
- **관계**: 숙소(M04 = *장소*·거주·케어) ──꾸미기──▶ 마이룸 꾸미기(M12 = *꾸미기 시스템*). **중복 아님, 역할 분리 + 연결**.
- **상태**: 🟡 (M12 참조)

## 3. 저택 등급 (Tier) — 시설 구성 변동 ★

> "저택에 따라 구성이 달라진다"(6/8). 등급업 = 방 수 + 실내 편의시설 확장.

| Tier | 명칭(예) | 방 | 시설 구성(예) |
|---|---|---|---|
| 1 | 오두막 | 2~3 | 주방·욕실·거실(기본) |
| 2 | 2층집 | +방 | + 연습실 |
| 3 | 저택 | 다수 | + GYM·재봉틀(공방)·수영장 등 |

- 🟠 등급 수·해금 조건·시설 슬롯 수 = 미정(§4)
- 6/5: "재증축" = 결제/자재로 등급업

## 4. 미결 · 쟁점 · 고려사항 (M04)

### 4-1. 미결
| ID | 안건 | 선택지 |
|---|---|---|
| L-1 | **저택 등급 수·해금** | 3단계 / 더 세분 / 자재 vs 결제 |
| L-2 | 시설별 효과 강도 | 강함(Fallout Shelter) / 단순 |
| L-3 | 방(육성 슬롯) 수 곡선 | 등급별 +n / 개별 증축 |
| L-4 | 마당(인벤토리) 표시 상한 | 전부 표시 / 페이지네이션 |
| L-5 | 의상 슬롯 수 | 8개 / Phase1 단순(3~4) |

### 4-2. 쟁점
| 쟁점 | A | B | 메모 |
|---|---|---|---|
| 시설 효과 | **강한 차등**(배치 효율↑) | **단순**(연출 위주) | 강하면 깊이↑·복잡도↑ |
| 마당 vs 방 | **방=육성 / 마당=대기** 이분 | 통합 컨트롤 | 6/5 이분 채택, UX는 통합 화면 |
| 케어 진입 | 방마다 개별 케어 | 숙소 대시보드 일괄 | 다마고치 정서 vs 효율 |

### 4-3. 고려사항
- **무대는 숙소에 없음** — 무대=구역(M06/M08). 숙소=케어·육성·거주.
- 저택 등급 = **메인맵 숙소 구역의 외관**에도 반영(지도에서 보임)
- 마당=인벤토리 ↔ 6/5 "갑판" 명칭 정리(마당/갑판 통일)
- **숙소(M04 = 장소) ──꾸미기──▶ 마이룸 꾸미기(M12 = 꾸미기 모듈) 연결** (중복 아님, 역할 분리). 명칭만 "마이룸/숙소" 일원화 검토
- VVNS House.jsx(754줄) 직이식 자산 활용 → 신규 시설만 추가
- 케어 = 숙소이므로 **다마고치 코어(M07) 시퀀스**가 여기서 구동

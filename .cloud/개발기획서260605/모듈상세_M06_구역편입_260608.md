# 🧩 M06. 구역·편입 (Zone) — 상세 명세 ★

> **위상**: 게임의 콘텐츠 그릇. 고정 15구역(앵커 2 + 편입 슬롯 13)에 **아이동 섬을 편입** → 미니게임·배치 생산·무대가 가동.
> **상태**: 🟢 구현 · ✅ 확정 · 🟡 기획중 · 🟠 미결

---

## 0. 개요
- **구역 = 고정 15**(앵커 2: **AREA-02 감정의파도항구 · AREA-13 꿈조각하우스(숙소)** + 편입 슬롯 13, 정서 풍경 불변). 배열은 `사이트맵_260611_최종확정.md`(L23) 기준.
- **편입**: 항해(M05)로 만난 아이동 섬 → 빈 슬롯에 채움 → 그 구역 가동.
- **구역 콘텐츠 3종**: ① 지정 미니게임(스킨) ② 아이동 배치 → 자동 생산 ③ (일부) 무대.
- **13슬롯 규칙**: 슬롯 점유는 1슬롯 1아이동. 초과 아이동은 삭제/이별 없이 숙소 마당·대기섬에 보관한다.

## 2. 화면별 상세 명세

### ZONE · 구역 슬롯 (편입 13 중 1)
- **영역**: 마이섬 구역(메인맵에서 탭)
- **목적**: 편입된 아이동의 콘텐츠 가동 공간.
- **진입**: 메인맵 구역 탭
- **상태별 표시**: 잠금(회색·구름) / 편입됨(아이동 점유) / 활성
- **이탈→다음**: 미니게임 / 배치 / (무대→M08)
- **상태**: ✅ (슬롯 구조) / 🟡 (콘텐츠)

### ZONE-MG · 지정 미니게임 (스킨형)
- **목적**: 아이동별 지정 게임 1종. **같은 게임 타입 + 아이동별 리소스/아이템만 차등**(스킨).
- **예(6/5)**: 황금멍·찌린양 둘 다 "카드 뒤집기" → 강아지/병아리 이미지·획득 아이템만 다름.
- **입장재**: 무소비 기본, 플레이 시작에는 아이동별 입장재(뼈다귀/하트 등) 가능.
- **상태**: ✅ 구조 확정 / 🟡 개별 게임 밸런스

### ZONE-PROD · 배치 → 자동 생산
- **목적**: 아이동을 구역에 배치 → 시간당 도감 아이템/자재 자동 생산.
- **효율**: 아이동 속성·케미로 차등(빨간 마크=비효율).
- **귀속**: 획득 도감 아이템 = 해당 아이동 귀속(M09).
- **상태**: ✅ (6/5)

### (무대 = 구역) → M08 데뷔
- 무대는 **각 편입 구역이 가질 수 있는 기능**이다. Phase1은 대표 1무대만 구현하고, 이후 구역별 무대를 확장한다.
- ✅🔒 **6/11 L26 확정**: 데뷔 동선 = 연습실 → **데뷔 회의실(SOOKSO-DEBUT, 결정·관리)** → `/stage` 대표 무대(개최). in-world 게이트 = **AREA-09 목표의반짝산**. AREA-01 꿈꾸는등대는 무대 아님(길잡이·온보딩 활동).
- 점수·연출·SUNO는 **M08**에서 관리한다.

## 3. 편입 → 구역 구성 (★ 핵심 설계)

> 아이동 섬(발견섬 = 페르소나 시각화 6구역)이 편입되면 마이섬 구역이 된다. 구성 결합 방식이 P-1.

| 구역 구성 요소 | 출처 | P-1 결정 |
|---|---|---|
| **정서 풍경 테마** | 마이섬 15구역(성장의정원…) 고정 | ✅ 유지 |
| **지정 미니게임** | 아이동 지정 게임 타입 + 스킨 | ✅ 아이동 레이어 |
| **배치 생산** | 아이동 배치 | ✅ 슬롯 점유 아이동 기준 |
| **무대(일부)** | 아이동 페르소나 무대 | ✅ 구역마다 가능, Phase1 대표 1무대 |

- **혼합안 확정**: 슬롯 = 정서 풍경(고정 정체성) + 그 위에 **편입 아이동 레이어**(미니게임 스킨·생산·무대). → "성장의정원" 슬롯에 황금멍 편입 = 정원 풍경 + 황금멍 카드뒤집기·생산.

### 3-1. 편입 절차 (LOCK)

| 단계 | 규칙 |
|---|---|
| 1. 항해 만남 | 보드에서 아이동 섬 조우 |
| 2. 영입 | 아이동 캐릭터가 숙소 마당/보유 목록에 합류 |
| 3. 슬롯 선택 | 유저가 빈 편입 슬롯 13개 중 선택. 추천 슬롯은 표시하되 강제하지 않음 |
| 4. 편입 완료 | 해당 슬롯에 아이동 레이어 생성, 미니게임/생산/무대 단서 활성 |
| 5. 교체 | 기존 점유 아이동은 대기섬으로 이동, 기록과 보유 아이템은 유지 |
| 6. 해제 | 슬롯만 비우며 생산/무대만 비활성. 이별/삭제 없음 |

### 3-2. 13슬롯 초과 규칙 (LOCK)

| 상황 | 처리 |
|---|---|
| 빈 슬롯 있음 | 유저가 슬롯을 선택해 즉시 편입 가능 |
| 13슬롯 모두 점유 | 새 아이동은 숙소 마당/대기섬에 보관 |
| 유저가 교체 선택 | 기존 슬롯 아이동은 대기섬으로 이동, 새 아이동이 슬롯 점유 |
| 대기 아이동 | 케어/도감/포토카드 기록은 유지, 구역 생산/무대만 비활성 |
| 삭제/이별 | POC 및 기본 정책에서 사용하지 않음 |

### 6-3. 고려사항
- 아이동 섬 = **발견섬 페르소나 시각화 6구역**(HERO of KINGDOM) → 편입되면 마이섬 구역 콘텐츠로 전환. 자산 재사용.
- 미니게임 = **타입 + 스킨** 절약 설계(766 개별 X)
- 배치 생산 도감 아이템 = 아이동 귀속(M09) → 진화 연료(M08)
- 무대 구역 = **데뷔 스테이지(M08)** 와 직결
- 안티골: Energy 강제·복잡 동선·헤비 그라인드

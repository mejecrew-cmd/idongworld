## 0. 30초 요약

**한 줄 정의** — 내 아이돌의 상징 동물(아이동)을 항해로 만나 마이섬에 들이고, 케어·연습으로 키워 무대에 세우는 **동물 페르소나 오마주 생활형 팬덤 육성 웹게임.**

**플랫폼** — React + Pixi.js (웹), 멀티세션, 딥링크 비목표.

**코어 루프**
```
항해(아이동 섬 만남·자원) → 영입(숙소 합류) → 편입(13슬롯 선택)
→ 숙소 케어·연습 → 도감템 수집 → 성장·데뷔(무대 구역) → 포토카드·공유 → 반복
```

## 4. 인사이클로피디아 (4장) — 데이터 시트 공급

> **여러 모듈에 공급되는 공통 스태틱 시트**의 집. 한 모듈이 소유하지 않고, 케어·도감·항해·성장이 다 같이 읽는다.
> *게임 안에서 어떤 화면/백과사전 UI로 보여줄지는 별개 — 여기는 순수 데이터 정의다.*

| 시트 | 내용 | 상태 |
|---|---|---|
| **아이동 시트** | 외형·생활·도감템 정의·speechStages (스태틱 마스터) | 정리 대상 — PoC `TBL-AIDONG-STATIC-SHEET` 참고 |
| **도감템 정의** | 도감템 마스터 (20/25칸 정합 필요) | [02 §D](02_처리목록_총목록.md) 정합 대기 |

> 정의(스태틱) = 4장 / 인스턴스(다이내믹, 유저 세이브) = 3장 각 모듈. 같은 "아이동"도 *종의 원본*(4)과 *내 개체의 현재값*(3)으로 갈린다.
> ✅🔒 **6/11 L28**: 게임 내 노출(인사이클로피디아 화면 ENCY-*)은 Phase1 = **읽기전용 정적 페이지**, **아이동 시트 000-999 ID 체계만 선행 락**. 마이 아이동 시트(M21, 다이내믹)에서 종 원본(ENCY-AIDONG)으로 연결.

# 2026년 6월 개발 로드맵

이 문서는 `.cloud/70_next_work_2026-06-12.md`의 작업 목록을 6월 내 마무리 목표로 재배열한 거시 일정표다.

최신 실행 보드는 `.cloud/83_next_work_2026-06-15_m1.md`다.

목표는 전체 게임 완성이 아니라, 6월 말까지 **기획과 코드가 같은 방향을 보게 만드는 POC 수직 루프**를 닫는 것이다.

## 6월 목표

6월 말 기준으로 다음 흐름이 끊기지 않아야 한다.

```text
로그인/타이틀
→ 오프닝 분기
→ 마이섬 15구역 허브
→ 숙소 케어
→ 항구/항해
→ 아이동섬 상륙
→ 아이동 영입
→ 13 편입 슬롯 선택
→ 구역 생산/도감템 placeholder
→ 마이룸 도감/콜렉션 확인
→ 데뷔 placeholder
```

## 6월 범위 원칙

- 6월은 `완성형 콘텐츠`가 아니라 `수직 루프와 데이터 경계`를 닫는다.
- 확정된 화면 route는 placeholder라도 앱 안에서 접근 가능해야 한다.
- M21 마이룸과 M22 아이동섬은 이름만 있는 모듈이 아니라 최소 route/screen/state 경계를 가져야 한다.
- 고정 15구역 + 13 편입 슬롯 구조를 코드와 문서의 기본값으로 맞춘다.
- 기존 세관, destination-shell-island, ship/lodge cargo 구현은 삭제하지 않고 compat/dev 자산으로 보존한다.
- BM, UGC, 소셜, 실시간 AI, SUNO 실제 연동은 6월 마감 범위에서 제외한다.

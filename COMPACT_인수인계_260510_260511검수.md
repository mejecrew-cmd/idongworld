## Backend Split Handoff Addendum 2026-05-24

- 백엔드 분리 Phase 1은 구현 완료 상태다. 하나의 Express 프로세스를 유지하되 route, service, repository, model 소유권은 분리되어 있다.
- MongoDB 연결은 로컬 개발 우선으로 구성되어 있으며, 이후 Atlas 전환은 `MONGO_URI` 변경으로 처리한다.
- 구현된 영속화 경계는 account/user, host state, my-aidong state, my-island state, codex state, route-neighbor state, ship state, zone state, 범용 module state, customs audit log를 포함한다.
- frontend 전환은 `VITE_SPLIT_STATE_SYNC=true`, `VITE_MODULE_ACTION_API_SYNC=true`를 기준으로 진행한다. `myIslandBootstrap`은 zone unlock/tutorial completion 후 `my-island` backend action API를 호출하고, `codexBootstrap`은 diary/slot/full registration 후 `codex` backend action API를 호출한다.
- zone 계열은 공통 backend action API를 가진다. `/api/modules/{zoneId}/collect`는 zone-local resource를 누적하고, `/api/modules/{zoneId}/clear`는 clear count/progress를 갱신한다.
- `zone-garden` frontend harvest는 DI hook으로 연결되어 있다. `GardenMiniGame`은 `configure({ onHarvest })`로 주입된 hook만 호출하고, `zoneGardenBootstrap`이 backend collect/clear API를 호출한다.
- `zone-oasis`, `zone-memory`, `zone-mine` frontend minigame은 각 모듈 package로 이전되었다. `MiniGameModal`은 module package에서 import하고, `zoneContentBootstrap`이 DI hook으로 backend collect/clear API 연결을 주입한다.
- `pnpm check:no-legacy-static`과 `pnpm check:no-legacy-runtime:local`은 2026-05-24에 통과했다. local Mongo, no-legacy backend, frontend runtime flag가 함께 검증된 상태다.
- `packages/modules/BACKEND_GUIDE.md`는 신규 모듈 API와 저장소 소유권을 다루는 모듈 백엔드 작업 지침이다.
- 남은 후속 작업은 production 수준의 customs adapter, transaction/session 검증 강화, frontend의 legacy state sync 의존 제거, 모듈별 business API 확장이다.

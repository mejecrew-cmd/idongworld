# 📥 데이터·자산 수신 사양 v1

> 개발은 콘텐츠 도착을 **기다리지 않는다**. 본 문서가 정의한 슬롯·스펙대로 들어오면 **즉시 게재 가능**한 상태로 유지.

---

## 0. 의도

```
[작가·디자이너] → 본 사양에 맞춘 파일 제출 → [PM] 검증·드롭 → [코드] 자동 인식
```

- **개발 진행**: 본 사양만 갖춰져 있으면 콘텐츠 0 인 상태에서도 빌드·타입체크 통과
- **수신 시 행동**: 사양 위반 → 즉시 반려 / 통과 → 디렉토리에 드롭 → 다음 빌드에 반영
- **자동화 시점**: Phase 1.5+ (현재는 PM 수동 드롭)

## 1. 수신 4 분류

| 분류 | 작업자 | 입력 형식 | 도착 위치 | 게재 형식 |
|---|---|---|---|---|
| 시나리오 | 작가 | outline `.md` (작성테이블 §9 빈 양식) | (PM 합의 채널) | `기획/시나리오/{type}_{id}.json` |
| 일기 | 작가 | 동일 | 동일 | `기획/시나리오/diary_{id}_day{N}.json` |
| 캐릭터 PNG | 디자이너 | PSD 또는 export PNG (R08 발주서 spec) | 디자이너 폴더 | `resources/aidong/{id}/{part}.png` |
| 사운드 | 큐레이터 | mp3/ogg 정규화된 (LUFS -14) | 큐레이터 폴더 | `resources/audio/{public,protected}/{id}.mp3` |

## 2. 시나리오 JSON 사양

### 2.1 명명 규칙
```
recruit_{scenarioId}.json     예: recruit_hwanggumeong.json
settle_{scenarioId}.json
cutscene_{설명}.json          예: cutscene_first_gacha.json
diary_{scenarioId}_day{N}.json  예: diary_hwanggumeong_day7.json
debut_{scenarioId}.json       (Phase 2)
```

`scenarioId` = shared-data/aidong-master/characters.csv 의 `scenarioId` 컬럼.

### 2.2 위치
```
기획/시나리오/
├─ _template_recruit.json      ← 빈 양식 (작성 시 복사)
├─ _template_settle.json
├─ _template_cutscene.json
├─ _template_diary.json
├─ recruit_*.json (5+편)
├─ settle_*.json (5+편)
├─ cutscene_*.json (1+편)
└─ diary_*.json (목표 100편)
```

### 2.3 스키마
- VN Scenario: `@idongworld/vn-runner/types` (Scenario·Scene·SceneChoice)
- Diary Fragment: 본 문서 §2.5

### 2.4 필수 필드 (VN Scenario)
```json
{
  "id": "recruit_hwanggumeong",      // 파일명과 동일
  "version": "1.0.0",
  "title": "황금멍 영입",
  "characters": ["황금멍"],
  "startScene": "s1",
  "metadata": {
    "writer": "작가명",
    "영감원_식별_검수": "0 매치 확인"
  },
  "scenes": { "s1": { ... } }
}
```

### 2.5 필수 필드 (Diary Fragment)
```json
{
  "id": "diary_hwanggumeong_day1",
  "version": "1.0.0",
  "characterId": "황금멍",
  "personaId": 56,
  "day": 1,
  "layer": "surface",          // surface (day1~9) / deep (day10~20)
  "metadata": {
    "tone": "한 줄",
    "trigger": "해금 조건",
    "motif": "한 단어",
    "writer": "작가명",
    "영감원_식별_검수": "0 매치 확인"
  },
  "fragments": {
    "wake": { "slot": "wake", "characterCount": N, "text": "..." },
    "morning": { ... },
    "noon": { ... },
    "evening": { ... },
    "night": { ... }
  }
}
```

### 2.6 검수 게이트 (자동)
[툴 리스트 §2.1.6](툴%20리스트_v1_260510_260511검수.md) 와 동일:
1. JSON Schema 통과
2. 길이 (라인 ≤200자·일기 슬롯 80~300자)
3. **영감원 grep 0 매치** (`bash tools/audit/checkPersonaLeak.sh <file>`)
4. 시그니처 워드 ≥1 (characters.csv 의 `signatureWords` 중)
5. 금칙어 0

## 3. 캐릭터 PNG 자산 사양

### 3.1 명명 규칙
```
resources/aidong/{characterId}/
├─ body.png                    ← 본체 (필수, protected)
├─ face_normal.png             ← 표정 5종 (필수, protected)
├─ face_happy.png
├─ face_surprised.png
├─ face_worried.png
├─ face_sleepy.png
├─ board_icon.png              ← 부루마블 보드 작은 아이콘 (필수, public — 회색 실루엣 도감용도 OK)
├─ signature.png               ← 데뷔 시그니처 (선택, protected, Phase 2)
└─ debut_bg.png                ← 데뷔 배경 (선택, Phase 2)
```

`characterId` = 한글 ID (황금멍·춤냥·양털곰·단풍볼·날카여우).
폴더는 사전에 모두 생성됨 (`.gitkeep` 보유).

### 3.2 해상도·포맷 (R07 사양서 SoT)
- 캐릭터 본체: **1024×1024 PNG (투명 배경)**
- 표정: **1024×1024 PNG**
- board_icon: **256×256 PNG**
- 발 baseline: y=920 (캔버스 상단 기준)
- 색공간: sRGB
- 압축: 디자이너 export 옵션 표준 (Phase 1.5+ tools/asset/optimize.ts 자동 webp 변환)

### 3.3 노출 정책 ([resources/README.md](resources/README_260511검수.md))
| 파일 | public | protected | 비고 |
|---|---|---|---|
| board_icon.png | ✅ | — | 도감 회색 실루엣 (영입 전) 포함 |
| body.png | — | ✅ | 영입 후 fetch (Phase 1.5+ backend stream) |
| face_*.png | — | ✅ | 동일 |
| signature.png | — | ✅ | 데뷔 후 |

### 3.4 수신 시 PM 작업
1. `resources/aidong/{characterId}/` 폴더에 디자이너 export 파일 드롭
2. 파일명 정합 확인 (위 3.1)
3. dev 빌드 시 즉시 AidongSprite 가 인식
4. **현재 dummy PNG (assets-dummy/) 우선순위 자동 폴백** — 누락 시 dummy 사용

## 4. 사운드 자산 사양

### 4.1 명명 규칙
```
resources/audio/
├─ public/
│  ├─ bgm_*.mp3       ← 공통 BGM (예: bgm_island.mp3)
│  └─ sfx_*.mp3       ← 공용 SFX (예: sfx_click.mp3)
└─ protected/
   ├─ bgm_signature_황금멍.mp3   ← 캐릭터 시그니처 BGM (영입 후 해금)
   └─ sfx_voice_*.mp3            ← 캐릭터 음성 SFX (Phase 2)
```

ID 카탈로그: `packages/frontend/src/data/audio.ts` (BGM 17·SFX 14)

### 4.2 포맷·정규화
- mp3 또는 ogg
- LUFS -14 (음량 표준화)
- 길이: BGM 60~180초 loop / SFX ≤2초
- 비트레이트: 128kbps (BGM) / 96kbps (SFX)

### 4.3 활성 시점
- Phase 1.5 (Week 6~7): Howler.js 통합 + BGM 첫 적용
- 1주차에는 폴더만 준비 — placeholder 상태 OK

## 5. 마스터 데이터 (CSV·JSON) 사양

| 위치 | 파일 | 편집자 | 용도 |
|---|---|---|---|
| `shared-data/aidong-master/` | characters.csv·json | PM | 본진 5명 마스터 (✅ 5/10) |
| `shared-data/cutscene-catalog/` | scenarios.csv 24·diaries.csv 100 | PM | ✅ Phase 1.5-15 완료 |
| `shared-data/species/` | 5종 + characterToSpecies | PM | ✅ Phase 1.5-15 완료 |
| `shared-data/schedule/` | 5존 시간대 + diarySlotMapping | PM | ✅ Phase 1.5-15 완료 |
| `packages/modules/{id}/balance.csv` | 모듈별 | PM·QA | 밸런스 파라미터 (Vite ?raw 런타임 로드) |
| `packages/modules/{id}/customs.csv` | 모듈별 | 모듈 작가 | 자원 변환 룰 |

**규칙**:
- 인코딩: UTF-8 (BOM 없음)
- 종결: LF (\n)
- 첫 행: 헤더 (snake_case 또는 camelCase 일관)
- 주석: `#` 으로 시작하는 행 (CSV 파싱 시 무시)
- 변경 시 README changelog 갱신

## 5. Asset Sync — 3단계 진화 계획

### 5.1 1단계: Frontend public/assets symlink (Phase 1, 1주차)

**시점**: 5/11 발사 직후
**방식**:
```bash
cd packages/frontend
ln -s "../../../../기획/와이어프레임/assets-dummy" public/assets
```
→ Vite는 symlink 통해 자산 정상 서빙·빌드 처리.

**장점**:
- 명령 1번. 부담 0.
- Drive 자산 변경 즉시 반영
- 백엔드 의존 X — frontend dev만 켜면 됨

**한계**:
- Backend가 자산 알지 못함 (frontend 단독)
- Vercel 배포 시 symlink는 처리됨 (단, 절대 경로면 깨짐 — 상대 경로 필수)

**적용 기간**: 1주차 (5/11~17). Backend Auth·DB 셋업 전.

---

### 5.2 2단계: Backend Express 정적 서빙 (Phase 1.5, 2~3주차)

**시점**: 5/14 목 (Backend Auth·DB 셋업 직후) 또는 5/18 월 (Week 2 시작)
**전환 작업**:
1. Frontend symlink 제거: `rm packages/frontend/public/assets`
2. Backend Express에 정적 서빙 1줄:
```typescript
// packages/backend/src/app.ts
import path from 'path'

const ASSET_ROOT = path.resolve(
  __dirname, '../../../../../기획/와이어프레임/assets-dummy'
)
app.use('/assets', express.static(ASSET_ROOT))
```
3. Frontend 자산 fetch URL 변경:
```typescript
// 1단계: <img src="/assets/캐릭터/...">
// 2단계: <img src="http://localhost:4000/assets/캐릭터/...">
//        (env: VITE_ASSET_URL=http://localhost:4000)
```

**장점**:
- Frontend·Backend 모두 자산 알 수 있음
- API + 자산 통합 운영 (production-like)
- Drive 자산 변경 여전히 즉시 반영

**한계**:
- Backend up 필수 — frontend dev 시 backend도 켜야 (concurrently로 가능)
- 절대 경로가 Drive 위치에 의존 — 환경 변수로 추출 필요

**적용 기간**: Week 2~7. 베타 빌드 직전까지.

---

### 5.3 3단계: tools/sync-assets.js build 시 복사 → CDN (Phase 3, 베타)

**시점**: 6/16 월 (Week 8 베타 빌드 시작)
**전환 작업**:
1. `tools/sync-assets.js` 작성 — Drive `assets-dummy/` → frontend `dist/assets/` 복사
2. Vercel 배포 시 dist 포함
3. Cloudflare CDN 자산 캐시
4. Backend는 자산 서빙 안 함 (API only)

**장점**:
- 빌드 결과 stand-alone (Drive 의존 X)
- CDN 글로벌 캐시
- Backend 부담 ↓

**한계**:
- Drive 자산 변경 시 sync 명령 재실행 + 재빌드
- 스크립트 작성 부담 (단순)

**적용 기간**: 베타 출시 이후 영구.

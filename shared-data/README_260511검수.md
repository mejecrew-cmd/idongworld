# 📊 shared-data — 공유 스태틱 데이터 풀

> 모듈 종속 X · 여러 모듈이 공통 참조하는 마스터 메타.

---

## 0. 위치

```
v1_260509/
├─ packages/         ← 코드 모듈
├─ resources/        ← 자산 풀 (PNG·webp·sound)
└─ shared-data/      ← (여기) 스태틱 마스터 메타
```

→ `packages/` 외부에 둠 (모듈 종속 회피·중앙 관리).

---

## 1. 카테고리

| 카테고리 | 역할 | 형식 |
|---|---|---|
| `aidong-master/` | 본진 5명 페르소나·파츠·시그니처·표정 정의 | CSV·JSON |
| `cutscene-catalog/` | 모든 컷신 정의·호출 지점 카탈로그 | JSON |
| `species/` | 5종 마스터 (강아지·고양이·곰·햄스터·여우) | CSV |
| `schedule/` | 5존 시간대 정의 (새벽·아침·오후·저녁·밤) | CSV |

---

## 2. 사용 방법

### 코드에서 참조
```ts
// 빌드 시점 import (Vite resolveJsonModule)
import characters from '../../shared-data/aidong-master/characters.json'

// 또는 fetch (런타임)
const res = await fetch('/shared-data/aidong-master/characters.json')
```

### 모듈 manifest 에서 명시
```ts
// ContentManifest.requires 에는 '시스템'만 — shared-data 는 implicit.
// 단 자료 의존이 핵심이면 description 에 명시 권장.
```

---

## 3. 책임 분담

- **수정 권한**: PM (본진 5명·컷신 카탈로그) + 디자이너 (파츠·표정)
- **검증**: `tools/` 스크립트로 정합성 검사 (예: 5명 모두 표정 5종 갖춤)
- **i18n**: 텍스트 키는 모듈 i18n namespace 로 → 여기는 ID·메타만

## 4. 최소 검증 명령

shared-data, i18n, 영감원 식별어, cutscene catalog, aidong-master mirror를 함께 확인할 때는 다음 명령을 우선 사용한다.

```bash
pnpm check:data
```

현재 포함 항목:

- `pnpm i18n:check`
- `pnpm audit:persona --silent`
- `pnpm audit:cutscene-catalog`
- `pnpm shared-data:check-aidong`

실제 `기획/시나리오` 루트까지 검사해야 할 때는 `SCENARIO_ROOT`와 `--require-files`를 사용해 scenario/diary catalog 검사를 별도로 강제한다.

---

## 5. 주의

- ❌ 모듈별 임시 데이터 → 모듈 폴더 자체 (예: `packages/modules/zone-garden/data/`)
- ❌ 자산 파일 (PNG·webp) → `resources/`
- ❌ 사용자별 동적 상태 → backend store
- ✅ 여러 모듈이 공통 참조하는 정적 메타만

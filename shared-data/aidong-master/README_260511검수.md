## 2. 필드 사양

| 필드 | 타입 | 설명 |
|---|---|---|
| `id` | string (한글) | 캐릭터 ID — 게임 내 표시·시나리오 character.id |
| `scenarioId` | string (영문 lowercase) | 시나리오 파일명 영문 — `recruit_{scenarioId}.json` |
| `species` | string (한글) | 종: 강아지·고양이·곰돌이·햄스터·여우 |
| `personaNumber` | number | 페르소나 # (페르소나리스트.csv 라인) |
| `gachaPool` | string | 가챠 풀 ID — `first-meeting` (Phase 1) |
| `assetCategory` | string | resources/{category}/ — 'aidong' |
| `description` | string | 한 줄 정수 (작가·디자이너용) |
| `signatureWords` | string[] | 자주 쓰는 표현 — 일기 검수 5번 게이트 |
| `voicePattern` | string | 말투 키워드 (`/` 구분) |
| `avoidPattern` | string | 피해야 할 톤 |
| `boardIconExposure` | 'public'\|'protected' | 보드 아이콘 노출 정책 |
| `bodyExposure` | 'public'\|'protected' | body·face 노출 정책 (영입 후 protected→public) |
| `defaultExpression` | string | 기본 표정 — normal·happy·surprised·worried·sleepy |

## 7. 자동 동기화 명령

```bash
pnpm shared-data:sync-aidong
pnpm shared-data:check-aidong
```

- `sync-aidong`: `characters.csv`를 기준으로 `characters.json`을 재생성한다.
- `check-aidong`: 현재 `characters.json`이 CSV에서 생성한 결과와 같은지 검사한다.
- 검사 실패 시 먼저 `sync-aidong`을 실행하고, 변경된 JSON을 확인한다.

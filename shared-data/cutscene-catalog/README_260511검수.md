## 1. scenarios.csv — 필드

| 필드 | 타입 | 설명 |
|---|---|---|
| `scenarioId` | string | 시나리오 ID (파일명 = `{type}_{ID 또는 설명}.json`) |
| `type` | enum | recruit·settle·cutscene·debut |
| `callSite` | string | 호출 지점 (cutscene-runner callSiteId 호환) |
| `characterIds` | csv | 등장 캐릭터 (한글, 다중 시 `;` 구분) |
| `status` | enum | written·planned·in-review |
| `writer` | string | 작성자 (PM·작가명·TBD) |
| `expectedDelivery` | date or phase | 'YYYY-MM-DD' 또는 'Phase 1.5' 등 |
| `length` | string | 'N scene' 형태 |
| `note` | string | 추가 메모 |

### 1.1 status 워크플로
```
planned → in-review → written
```

## 2. diaries.csv — 필드

| `diaryId` | string | `diary_{scenarioId}_day{N}` |

## 3. 자동 정합 검사

```bash
# scenarios.csv 구조와 written 시나리오 정합
pnpm audit:scenario-catalog

# diaries.csv 구조와 written 일기 정합
pnpm audit:diary-catalog

# 둘 다 실행
pnpm audit:cutscene-catalog
```

현재 워크스페이스에 실제 `기획/시나리오` 폴더가 없으면 구조 검증만 수행하고 경고를 낸다. 실파일 존재까지 강제하려면 `SCENARIO_ROOT`를 지정하고 `--require-files`를 사용한다.

```bash
SCENARIO_ROOT=../../기획/시나리오 python tools/audit/checkScenarioCatalog.py --require-files
SCENARIO_ROOT=../../기획/시나리오 python tools/audit/checkDiaryCatalog.py --require-files
```

검증 기준:

- `written` 상태는 실파일 루트가 있을 때 `{scenarioId}.json` 또는 `{diaryId}.json` 파일이 있어야 한다.
- `planned`, `in-review` 상태는 실파일이 없어도 통과한다.
- `scenarios.csv`는 필수 컬럼, 중복 ID, type/status 허용값, characterIds 정합을 검사한다.
- `diaries.csv`는 필수 컬럼, 중복 ID, characterId/personaNumber/day/layer/status 정합을 검사한다.

→ 본 카탈로그가 **계획·진척표** 역할 + **자동화 검증의 기준**.

## 4. 운영

| 시점 | 누가 | 행동 |
|---|---|---|
| 작가 outline 도착 | PM | status `planned → in-review` |
| JSON 변환·검수 통과 | PM | status `in-review → written` + writer·expectedDelivery 갱신 |
| 매주 금요일 | PM | written 합계·페이스 점검 → COMPACT 인수인계 갱신 |
| Phase 단계 전환 | PM | planned 잔여 → 다음 Phase 일정으로 재할당 |
